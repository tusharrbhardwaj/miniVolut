# miniVolut
A step-by-step Flask learning project that demonstrates how frontend, backend, databases, and deployment work together by building a complete Python web application from scratch.

The purpose of this repository is **not** to build a production-ready application immediately, but to understand how every component of a web application communicates with one another—from the browser to the server, database, and back again.

This project serves as the foundation for larger projects (such as DBCrypt) by focusing on understanding rather than simply writing code.

---

## Objective

The primary objective of this project is to answer questions such as:

- How does a browser communicate with a backend?
- What actually happens when a user clicks a button?
- How does Flask receive requests?
- How does Python communicate with a database?
- How are HTML pages generated dynamically?
- What role does JavaScript play?
- How are web applications deployed to a Linux server?

Rather than learning these topics separately, MiniVault connects them into one complete workflow.

## Why This Repository Exists

Most tutorials teach Flask, databases, HTML, JavaScript, Linux, or deployment independently.

This repository is an attempt to learn how all of these technologies work **together** as one complete system.

The focus is on understanding:

```
Browser
    ↓
HTTP Request
    ↓
Flask
    ↓
Python
    ↓
Database
    ↓
Python
    ↓
Flask
    ↓
HTML Response
    ↓
Browser
```

Understanding this request-response cycle is the primary goal of the project.
